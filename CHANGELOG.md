# 更新日志

## 1.1.4 (2026-07-16)

- 改进延迟搜索构建的安全性：加入重载代次和旧结果丢弃逻辑。
- 加入 EMI 重载/后台搜索状态锁，避免 EMI 清理内部数据时后台搜索仍在读取。
- 加入客户端关闭处理，减少退出时后台搜索构建刷错。
- 移除对 EMI 私有字段 `searchedStacks` 的直接写入，改为按 EMI 原流程刷新搜索。
- 缓存保存改为先写临时文件，再替换旧缓存，降低坏缓存概率。

## 1.1.3 (2026-06-01)

- 修复缓存命中时，带数据组件的物品栈显示为 `<Missing>` 的问题，例如 Ender IO conduit。
- 改进后台搜索构建完成后的搜索结果同步。
- 内部 EMI 字段访问改为 `VarHandle`。

## 1.1.2 (2026-05-24)

- 缓存格式从 JSON 改为 StreamCodec + LZ4 二进制格式，体积更小，读写更快。

## 1.1.1 (2026-05-17)

- 添加聊天提示开关、调试开关和加速总开关。
- 添加本地化支持。
- 配置存储改为线程安全的 key/value 文本文件。
- 统一聊天输出逻辑到 `ChatHelper.sendIfNotHidden()`。

## 1.1.0 (2026-05-09)

- 许可证从 GNU LGPL 3.0 切换为 GNU AGPL 3.0。

## 1.0.1 (2026-05-07)

- 修复缓存命中时 EMI 合成记录/使用记录丢失的问题；缓存命中后仍会执行必要的后续重载阶段。

## 1.0.0 (2026-05-07)

- 初始版本：缓存 `EmiStackList.stacks`。
- 添加延迟搜索索引构建。
- 添加模组列表哈希校验和缓存失效机制。
- 添加重载耗时诊断。
- 添加 `/emiacc status`、`/emiacc clear`、`/emiacc reload` 和 `/emiacc reload --force` 指令。
